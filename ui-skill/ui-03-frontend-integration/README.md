# 模块：前端集成

## 定位

前后端联调与上线的桥梁。将前端代码与后端API对接，完成构建部署和性能优化，确保产品可上线。目标是**通过契约驱动和自动化保障，实现前后端无缝联调和高质量上线**。

## 何时使用

- UI前端代码已生成，需要与后端API联调
- 后端API契约已定义，需要生成前端请求层
- 需要配置前端构建、部署和CI/CD流水线
- 需要优化前端性能，确保上线质量

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 前端集成 | frontend-integration-orchestrator | 协调API消费、构建部署和性能优化 | 需要前后端联调集成和上线时 |

## Pipeline Skill 清单

### API契约消费（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| api-contract-consume | 基于OpenAPI生成前端请求层、类型定义、Mock数据和Hook | API契约文档、页面数据需求 | 请求代码+类型+Mock |

### 构建部署（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| frontend-build-deploy | 生成构建配置、环境管理、CDN策略和CI/CD流水线 | 项目信息、部署目标 | 构建配置+CI/CD |

### 性能优化（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| frontend-performance | 分析性能瓶颈，生成包体积/加载/渲染优化方案 | 前端代码、构建产物、性能数据 | 优化方案+性能预算 |

## 执行顺序

```
阶段1            阶段2              阶段3
┌──────────┐  ┌──────────────┐  ┌──────────────┐
│ API消费   │→│ 构建部署      │→│ 性能优化      │
│ 请求层+Mock│  │ CI/CD+CDN    │  │ 包体积+加载   │
└──────────┘  └──────────────┘  └──────────────┘
```

- API消费先行，Mock数据支持前后端并行开发
- 构建部署配置CI/CD自动化流水线
- 性能优化确保上线质量达标

## 输出路径

```
output/ui-frontend-integration/
├── api-contract-consume/
├── frontend-build-deploy/
└── frontend-performance/
```

## 阶段卡口

### 进入构建部署前需满足：
- 100%接口有类型定义+Mock数据

### 进入性能优化前需满足：
- 构建成功+CI流水线通过

### 上线前需满足：
- LCP≤2.5s
- 首屏JS≤200KB
- P0性能问题=0

## 人类决策点

| 决策点 | 说明 |
|--------|------|
| API契约确认 | AI消费API契约后，人类确认接口理解是否正确 |
| 部署目标选择 | 人类确认部署平台和环境配置 |
| 性能预算调整 | 人类确认性能预算阈值是否合理 |

## 核心信念

- 前后端通过契约解耦，集成通过自动化保障
- Mock先行，后端未就绪不阻塞前端开发
- 性能预算写入CI，超标自动拦截
- 构建可复现，回滚秒级
