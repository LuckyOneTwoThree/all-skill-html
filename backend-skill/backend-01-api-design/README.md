# 模块：API设计

## 定位

后端开发的起点。基于PRD和数据模型设计API契约、安全策略和认证鉴权方案，实现前后端契约驱动开发。目标是**先契约后实现，前后端基于契约并行开发**。

## 何时使用

- 产品需求已明确，需要设计后端API接口
- 需要定义API契约供前端并行开发
- 需要设计API安全策略和限流规则
- 需要设计用户认证和权限管理方案

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| API设计 | api-design-orchestrator | 协调API契约、安全和认证鉴权的完整设计 | 需要设计后端API时 |

## Pipeline Skill 清单

### API契约设计（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| api-contract | 设计RESTful/GraphQL接口契约，生成OpenAPI 3.0规范 | PRD、数据模型 | openapi.yaml |

### API安全设计（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| api-security | 设计限流规则、数据加密、输入校验、CORS和安全头 | API契约、安全等级 | 安全策略文档 |

### 认证鉴权设计（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| auth-design | 设计JWT/OAuth2/SSO认证、RBAC/ABAC权限、多租户隔离 | PRD、API契约 | 认证鉴权方案 |

## 执行顺序

```
阶段1            阶段2              阶段3
┌──────────┐  ┌──────────────┐  ┌──────────────┐
│ API契约   │→│ API安全       │→│ 认证鉴权      │
│ 接口+版本 │  │ 限流+加密+校验│  │ JWT+权限+租户 │
└──────────┘  └──────────────┘  └──────────────┘
```

- API契约是基础，安全和鉴权基于契约设计
- 安全内建而非外挂，与契约同步设计
- 认证鉴权统一方案，不每个接口单独处理

## 输出路径

```
output/backend-api-design/
├── api-contract/
├── api-security/
└── auth-design/
```

## 阶段卡口

### 进入API安全前需满足：
- 每个资源有CRUD定义+错误码体系

### 进入认证鉴权前需满足：
- 100%接口有安全级别+限流规则
- L4接口安全策略完整

### 进入下一模块（数据架构）前需满足：
- OpenAPI文档完整
- 认证方案+权限模型+会话管理完整

## 人类决策点

| 决策点 | 说明 |
|--------|------|
| API风格选择 | RESTful vs GraphQL，人类确认 |
| 安全等级确认 | 标准vs高安全，影响限流/加密/审计策略 |
| 权限模型选择 | RBAC vs ABAC，影响权限管理复杂度 |
| 多租户策略 | 隔离级别影响成本和安全，人类确认 |

## 核心信念

- 契约驱动开发，前后端基于契约并行
- 安全内建而非外挂
- 默认拒绝，未明确允许的一律拒绝
- 版本管理从第一天开始
