# 模块：数据架构

## 定位

后端数据层的设计核心。基于业务需求设计数据模型、缓存策略和迁移方案，确保数据存储合理、访问高效、变更安全。目标是**数据模型决定系统上限，缓存策略决定性能下限**。

## 何时使用

- API契约已设计，需要定义数据存储结构
- 需要设计ER模型、表结构和索引策略
- 需要评估分库分表方案
- 需要设计缓存架构和一致性策略
- 需要规划数据库Schema迁移方案

## 子模块与编排器

| 子模块 | 编排器 | 作用 | 何时调用 |
|--------|--------|------|----------|
| 数据架构 | data-architecture-orchestrator | 协调数据模型、缓存策略和迁移方案 | 需要设计数据层架构时 |

## Pipeline Skill 清单

### 数据模型设计（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| data-model | 设计ER模型、表结构、索引策略和分库分表方案 | PRD、API契约、数据量预估 | DDL+数据字典+ER图 |

### 缓存策略设计（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| cache-strategy | 设计多级缓存架构、一致性策略和穿透/击穿/雪崩防护 | 数据模型、API契约、并发量预估 | 缓存方案+Key规范+监控 |

### 数据迁移方案（1个）

| Skill | 作用 | 输入 | 输出 |
|-------|------|------|------|
| data-migration | 设计Schema迁移、数据迁移和回滚方案 | 当前Schema、目标Schema、数据量 | 迁移脚本+回滚脚本+校验方案 |

## 执行顺序

```
阶段1            阶段2              阶段3
┌──────────┐  ┌──────────────┐  ┌──────────────┐
│ 数据模型  │→│ 缓存策略      │→│ 数据迁移      │
│ ER+DDL   │  │ 多级+一致性   │  │ 迁移+回滚     │
└──────────┘  └──────────────┘  └──────────────┘
```

- 数据模型是基础，缓存和迁移都依赖模型设计
- 缓存按需设计，有性能瓶颈才加缓存
- 迁移方案确保每个变更可回滚

## 输出路径

```
output/backend-data-architecture/
├── data-model/
├── cache-strategy/
└── data-migration/
```

## 阶段卡口

### 进入缓存策略前需满足：
- ER图+DDL+数据字典完整

### 进入数据迁移前需满足：
- 穿透/击穿/雪崩防护全覆盖

### 进入下一模块（后端架构）前需满足：
- 100%变更有回滚脚本
- 数据模型人类已确认

## 人类决策点

| 决策点 | 说明 |
|--------|------|
| 范式vs反范式 | 读写比决定，人类确认平衡点 |
| 分库分表策略 | 影响成本和复杂度，人类确认 |
| 缓存一致性级别 | 强一致vs最终一致，人类确认 |
| 迁移执行时间 | 低峰期窗口，人类确认 |

## 核心信念

- 数据是系统根基，模型决定上限
- 索引精准，每个索引有查询场景支撑
- 缓存有据，一致性显式定义
- 迁移可回滚，数据不丢失
